# Семейный бюджет

PWA для учёта семейных доходов и расходов. Работает на iOS и Android как
устанавливаемое приложение (Progressive Web App). Один общий семейный бюджет,
без отдельных счетов.

## Возможности

- **Общий бюджет на двоих** — два пользователя (Муж/Жена) со своими PIN, видно,
  кто внёс запись.
- **Ручной ввод** расходов с категориями (категории можно добавлять/удалять).
- **Разбор через Claude (Opus)** — сфотографируйте чек или опишите трату словами,
  Claude определит сумму и категорию, вы подтверждаете и сохраняете.
- **Ежемесячные платежи** — регулярные траты, которые можно провести в один тап.
- **Коммуналка** — 4 обязательных платежа (за газ, воду, электричество, мусор),
  которые нужно заполнять каждый месяц до 20 числа. Для газа/воды/электричества
  объём обязателен, для мусора — нет. После ввода автоматически считается
  разница с предыдущим периодом. На главной — предупреждение, если после 20 числа
  не всё заполнено.
- **Статистика** за месяц: сумма и разбивка по категориям.

## Архитектура

```
Finance/
├── backend/      Node.js + TypeScript, Fastify, SQLite (better-sqlite3)
│                 Claude Agent SDK (headless, авторизация подпиской Max)
├── frontend/     Ionic React + Vite, PWA (manifest + service worker)
├── Dockerfile    multi-stage: сборка фронта и бэка → бэкенд отдаёт статику + API
└── docker-compose.yml
```

Бэкенд отдаёт и API (`/api/*`), и собранный фронтенд, поэтому наружу торчит
один порт (3000). Данные (SQLite + загруженные чеки) лежат в томе `/data`.

## Как Claude использует вашу подписку Max

Разбор чеков и сообщений идёт через **Claude Agent SDK** (`@anthropic-ai/claude-agent-sdk`),
который авторизуется тем же способом, что и Claude Code — по OAuth-токену вашей
подписки Max. Обычный Anthropic API (с оплатой по токенам) **не** используется.

Токен получается один раз командой на вашей машине (где установлен Claude Code):

```bash
claude setup-token
```

Команда выведет токен вида `sk-ant-oat...`. Его нужно положить в `.env` в переменную
`CLAUDE_CODE_OAUTH_TOKEN` (см. ниже). Токен долгоживущий, но не вечный — если разбор
чеков перестанет работать, перевыпустите его той же командой и обновите `.env`.

> Оговорка: использование подписки Max для серверного приложения — неофициальный
> сценарий. Действуют лимиты подписки. Если понадобится стабильность/масштаб —
> можно переключиться на обычный Anthropic API (это отдельная оплата).

## Локальный запуск (разработка)

Нужен Node.js 20+.

```bash
# 1. Бэкенд
cd backend
npm install
CLAUDE_CODE_OAUTH_TOKEN=sk-ant-oat... npm run dev     # http://localhost:3000

# 2. Фронтенд (в отдельном терминале)
cd frontend
npm install
npm run dev                                            # http://localhost:5173
```

Dev-сервер фронтенда проксирует `/api` и `/uploads` на бэкенд (порт 3000).
При первом запуске бэкенд создаёт БД и двух пользователей с PIN `1234`.

## Запуск в Docker

```bash
cp .env.example .env
# отредактируйте .env: задайте JWT_SECRET и вставьте CLAUDE_CODE_OAUTH_TOKEN

docker compose up -d --build
```

Приложение будет на `http://<хост>:3000`. Данные сохраняются в docker-томе
`finance-data` и переживают пересборку контейнера.

Обновление после изменений в коде:

```bash
docker compose up -d --build
```

## Деплой на VPS

1. Установите Docker и Docker Compose на VPS.
2. Скопируйте проект на сервер (`git clone` или `scp`).
3. Создайте `.env` (см. выше). **Обязательно** задайте длинный случайный
   `JWT_SECRET` и вставьте `CLAUDE_CODE_OAUTH_TOKEN`.
4. `docker compose up -d --build`.

### HTTPS

Для начала можно работать по HTTP на IP. Но учтите: **PWA полноценно ставится
как приложение и использует камеру только по HTTPS** (или на `localhost`). По
голому HTTP на IP-адресе:

- камера для съёмки чека может быть недоступна (браузер блокирует `getUserMedia`
  и `capture` на небезопасном origin) — останется выбор файла из галереи;
- предложение «установить на домашний экран» может не появиться.

Варианты HTTPS:

- **Домен + Let's Encrypt** (рекомендуется). Поставьте перед приложением
  reverse-proxy (Caddy проще всего — сам получает сертификат):

  ```
  # Caddyfile
  ваш-домен.ru {
      reverse_proxy localhost:3000
  }
  ```

- **HTTPS на IP** — только с самоподписанным сертификатом. Браузеры будут ругаться,
  и на телефоне это неудобно; как временное решение годится, для постоянного —
  лучше домен.

Рекомендация: возьмите дешёвый домен и Caddy — тогда PWA, камера и установка на
телефон заработают полностью.

## Первый вход

- Пользователи: **Муж** и **Жена**.
- PIN по умолчанию — из `DEFAULT_PIN` (в примере `1234`).
- Поменять PIN: вкладка **Ещё → Сменить PIN**.

## Переменные окружения

| Переменная                | Назначение                                              |
| ------------------------- | ------------------------------------------------------- |
| `APP_PORT`                | Порт на хосте (по умолчанию 3000)                       |
| `JWT_SECRET`              | Секрет подписи токенов — обязательно поменять           |
| `DEFAULT_PIN`             | PIN пользователей при первом запуске                    |
| `CLAUDE_CODE_OAUTH_TOKEN` | Токен подписки Max (`claude setup-token`)               |
| `CLAUDE_MODEL`            | Модель Claude (по умолчанию `claude-opus-4-8`)          |
| `DB_PATH`                 | Путь к файлу SQLite (в Docker — `/data/finance.db`)     |
| `UPLOAD_DIR`              | Папка загруженных чеков (в Docker — `/data/uploads`)    |
